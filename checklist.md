# Checklist

- [x] 현재 루트 정적 사이트 구조를 확인한다.
- [x] Git 히스토리에서 이전 콘텐츠 관리 도구 위치를 확인한다.
- [x] GitHub Pages 배포 기준을 루트 정적 파일로 유지하기로 결정한다.
- [x] 제품 정보 원본 데이터 파일 위치를 확정한다.
- [x] 로컬 전용 콘텐츠 관리 도구를 현재 구조에 맞게 복구하거나 개편한다.
- [x] 도구에서 제품 추가, 수정, 숨김, 삭제 흐름을 지원한다.
- [x] 도구에서 루트 정적 HTML을 재생성할 수 있게 한다.
- [x] 생성된 루트 페이지를 로컬에서 검증한다.
- [x] 변경 내용을 커밋한다.
- [x] 콘텐츠 관리 도구를 더블클릭으로 실행하는 루트 스크립트를 추가한다.
- [x] 콘텐츠 관리 도구에서 회사 기본 정보를 편집할 수 있게 한다.
- [x] 콘텐츠 관리 도구에서 연혁과 기술 정보를 편집할 수 있게 한다.
- [x] 회사 정보 변경이 회사소개와 기술정보 정적 페이지에 반영되게 한다.
